# Summary Parser

`ConvertFrom-CoreCyclerSummaryText` in `helpers/CoreCyclerPhaseConfig.psm1` parses CoreCycler final summary text or log excerpts without running CoreCycler. It is offline and side-effect-free.

## Parsed Data

- `CoresWithErrors`: sorted physical core IDs listed in the final summary error section.
- `ErrorDetails`: per-core detail rows with core, CPU, date, error type, stress error, and optional following message.
- `Whea`: per-core WHEA counts and total WHEA count.
- `CurveOptimizer`: detected CO table, whether it was adjusted, core IDs, starting values, current values, and final values.

## Fixtures

The parser is covered by small synthetic fixtures:

- `tests/fixtures/final-summary-with-errors.log`
- `tests/fixtures/final-summary-clean.log`
- `tests/fixtures/final-summary-adjusted-clean.log`

These fixtures are intentionally small and model the output generated by `Show-FinalSummary`. Add new fixtures before changing parser behavior for new summary/log shapes.

## Example

```powershell
Import-Module .\helpers\CoreCyclerPhaseConfig.psm1 -Force

$text = Get-Content .\tests\fixtures\final-summary-with-errors.log -Raw
$summary = ConvertFrom-CoreCyclerSummaryText -Text $text

$summary.CurveOptimizer.FinalValues
$summary.Whea.TotalCount
$summary.CoresWithErrors
```
